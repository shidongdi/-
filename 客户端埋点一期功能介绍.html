客户端埋点一期功能介绍

客户端一期目标：

  1. 以SDK的方式，开发统计功能；
  2. SDK提供统计方法（点击事件、页面停留事件、自定义事件等），业务层通过调用SDK对应方法，埋点数据；
  3. 以sqlite数据库的方式，保存统计信息；
  4. 服务端和客户端定义统计数据上传协议；
  5. 创建本地服务，获取统计数据；
  6. 按条数或者按时间间隔分批将统计数据上传到服务端；

现阶段已实现的功能：

客户端信息搜集流程：埋点数据 -> 存储数据 -> 上传数据

功能结构图：

埋点数据：

​ 统一的埋点入口，通过TrackerManager管理搜集埋点的功能类，包括View埋点搜集(ViewTracker)、页面埋点搜集(PageTracker)、通用埋点搜集(CommonTracker)等，在合适的地方调用埋点接口，搜集信息；

TrackerManager.instance.viewTracker.onClick(view) // 搜集点击事件

TrackerManager.instance.pageTracker.onResume(context) // 搜集页面事件

搜集数据示例(点击):

存储数据：

​ 埋点事件，统一存储在sqlite数据库中，单条事件的数据结构：

class EventEntity {
    var id: Long           // 唯一id标识
    var eventType: String  // 事件类型
    var eventData: String  // 事件数据，可扩展(json)
    var userId: String     // 用户id
    var timestamp: Long    // 时间戳
}
上传数据：

​ 上传数据分为定时上传、立即上传两种： ​ 定时上传：创建定时器，指定时间间隔(duration)上传一定数量(number)的事件信息，duration和number可动态配置； ​ 立即上传：每产生一条数据，就立即上传，不进数据库，一般在测试模式下使用；

​ 上传流程图：

​

​ sdk提供数据的处理功能，具体的上传业务也可通过接口回调的方式实现；sdk同时也包括上传成功的数据清除和上传失败之后的重新上传。 ​ 数据的上传得格式(protobuf)：

message MessageContent {
    string eventType = 301; // 事件类型
    string eventData = 302; // 事件数据
    int64 timestamp = 303;  // 时间戳
    string userId = 304;    // 用户id
}

message Message {
    repeated MessageContent content = 200;
}

message MessageRequest {
    string uuid = 100; // 设备唯一标识
    Message message = 101;
}
